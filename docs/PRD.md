# PRD — 온라인 오목 (Omok Online)

> **버전**: 1.2  
> **작성일**: 2026-07-02  
> **상태**: MVP 완료 (공개방/랭킹전 ELO 추가)

---

## 1. 개요

### 제품 한 줄 설명
브라우저에서 바로 플레이할 수 있는 온라인 오목 게임. **렌주룰(Renju)**을 적용하며, 친구와 실시간 대전하거나 AI와 대국한다.

### 게임 규칙: 렌주룰 (Renju)
국제 오목 표준 규칙인 렌주룰을 적용한다. 흑(선수)에게만 다음 금수(禁手)가 적용되며, 금수 위치에 착수하면 즉시 패배한다.

| 금수 종류 | 설명 |
|---|---|
| **33 (삼삼)** | 한 수로 열린삼을 2개 이상 동시에 만드는 수 |
| **44 (사사)** | 한 수로 사(四)를 2개 이상 동시에 만드는 수 |
| **장목** | 6목 이상의 연속 돌 |

**거짓금수**: 삼·사를 완성하는 자리가 흑 입장에서 금수이면 진짜 삼·사로 인정하지 않는다 (거짓금수로 허용).  
백(후수)에게는 금수가 없으며 장목도 승리로 인정한다.

### 배경 및 목적
기존 오목 사이트들은 회원가입이 필요하거나 UI가 낡아 진입 장벽이 높다. 이 프로젝트는 링크 하나로 바로 대전을 시작할 수 있는 가벼운 웹 오목을 목표로 한다.

### 범위 (MVP)
- 실시간 1:1 온라인 대전 (비공개 방 코드 공유 / 공개방 목록)
- 랭킹전 자동 매칭 + ELO 레이팅
- AI 대전 (Minimax 알고리즘)
- 게임 중 채팅
- 턴 타이머

## 3. 기능 명세

### 3.1 로비 (Lobby)

3탭 구성: **비공개방 | 공개방 | 랭킹전**

| 기능 | 설명 | 상태 |
|---|---|---|
| 닉네임 입력 | 최대 12자, 선택 사항 (미입력 시 '플레이어' 자동 부여) | ✅ 완료 |
| 방 만들기 (비공개) | 6자리 랜덤 코드 발급, 상대방 대기 | ✅ 완료 |
| 방 입장 (비공개) | 코드 입력 후 입장, 입장 즉시 게임 시작 | ✅ 완료 |
| 공개방 목록 | 진행 가능한 공개방을 실시간 폴링으로 표시, 클릭 시 즉시 입장 | ✅ 완료 |
| 랭킹전 자동 매칭 | 매칭 큐 등록 → 대기 중인 상대와 자동 매칭 → 게임 시작 | ✅ 완료 |
| AI 대전 시작 | 서버 없이 클라이언트에서 AI와 즉시 대전 | ✅ 완료 |

### 3.2 게임 (Game)

| 기능 | 설명 | 상태 |
|---|---|---|
| 오목판 렌더링 | 15×15 Canvas 보드, 화점 표시 | ✅ 완료 |
| 돌 놓기 | 클릭으로 착수, 마지막 수 강조 표시 | ✅ 완료 |
| 5목 승리 판정 | 가로/세로/대각선 4방향 판정 | ✅ 완료 |
| 무승부 판정 | 보드 전체 채워지면 무승부 | ✅ 완료 |
| 금수 판정 (렌주룰) | 흑의 33/44/장목 착수 시 즉시 패배 처리 | ✅ 완료 |
| 거짓금수 허용 | 완성 자리가 금수인 삼·사는 진짜 금수로 인정 안 함 | ✅ 완료 |
| 금수 위치 표시 | 흑 차례 시 금수 자리에 빨간 삼각형(▲) 표시 | ✅ 완료 |
| 턴 표시 | 현재 차례 플레이어 강조 | ✅ 완료 |
| 마지막 수 표시 | 직전 착수 위치에 색상 테두리 표시 | ✅ 완료 |
| 타이머 | 턴당 3분, 초과 시 자동 패배 | ✅ 완료 |
| 항복 | 온라인 모드에서 항복 선택 가능 | ✅ 완료 |
| 재경기 | 양쪽 동의 시 재시작 (온라인) / 즉시 재시작 (AI) | ✅ 완료 |
| 접속 끊김 처리 | 상대방 연결 끊기면 자동으로 게임 종료 | ✅ 완료 |

### 3.3 채팅

| 기능 | 설명 | 상태 |
|---|---|---|
| 실시간 채팅 | 게임 중 방 내 메시지 전송/수신 | ✅ 완료 |

| 자동 스크롤 | 새 메시지 수신 시 하단으로 자동 이동 | ✅ 완료 |

### 3.4 AI 엔진

| 기능 | 설명 | 상태 |
|---|---|---|
| 즉시 승리 | 한 수로 이길 수 있으면 즉시 착수 | ✅ 완료 |
| 즉시 방어 | 상대 4목 차단 | ✅ 완료 |
| Minimax depth-3 | 3수 앞 탐색 + alpha-beta pruning | ✅ 완료 |

### 3.5 랭킹전 / ELO 레이팅

| 기능 | 설명 | 상태 |
|---|---|---|
| ELO 레이팅 | 시작 레이팅 1200, K=32로 승/패/무 후 자동 갱신 | ✅ 완료 |
| 레이팅 영구 저장 | `server/data/ratings.json` 파일에 저장 (서버 재시작에도 유지, 단 익명 브라우저 UUID 기준이라 기기 변경 시 전적 이어지지 않음) | ✅ 완료 |
| 레이팅 변화 표시 | 게임 종료 모달에 레이팅 변화량 표시 (예: +32 → 1232) | ✅ 완료 |
| 순위표 | 상위 20명 랭킹 페이지 | ✅ 완료 |

## 4. 미구현 / 향후 과제

| 기능 | 우선순위 | 설명 |
|---|---|---|
| 관전 모드 | 중간 | 진행 중인 방 관전 |
| 회원가입/로그인 | 낮음 | 현재는 localStorage UUID로 익명 식별. 기기 변경 시 전적이 끊기므로 계정 기능 필요 |
| AI 난이도 선택 | 중간 | 쉬움(depth-1) / 보통(depth-3) / 어려움(depth-5) |
| 모바일 최적화 | 중간 | 터치 이벤트, 반응형 보드 크기 |

## 5. 비기능 요구사항

| 항목 | 목표 |
|---|---|
| 응답 속도 | 착수 이벤트 → 상대 화면 반영 < 100ms (LAN 기준) |
